# SYSTEM_OVERVIEW

## Цель
Собрать практичную и экономную систему оркестрации разработки:
- общение и управление задачами через OpenClaw (Telegram)
- выполнение кодовых изменений через локальный Claude Code пользователя
- минимизация токенов и сложности

## Базовая архитектура (v1)
1. **OpenClaw (оркестратор):**
   - принимает тикет/задачу в Telegram
   - уточняет требования
   - формирует план и структурированный payload
   - запускает нужный режим выполнения через bridge
2. **Bridge backend (тонкий слой):**
   - HTTP endpoint для передачи задач в Claude Code
   - режимы: `implement`, `review`, `tests`
   - возвращает краткий результат + ссылки/идентификаторы артефактов
3. **Claude Code (локально у пользователя):**
   - вносит изменения в код
   - выполняет проверки
   - работает в отдельной ветке
   - формирует PR

## Что НЕ делаем на старте
- не строим «зоопарк» агентов
- не делаем полную автономию без контроля
- не подключаем серые схемы токенов Claude Max в OpenClaw

## Принципы
- Human-in-the-loop на ключевых шагах
- PR-only, без прямых изменений в `main`
- сначала MVP, потом масштабирование
- «меньше магии, больше предсказуемости"
