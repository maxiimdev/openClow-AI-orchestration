# FLOW

## Целевой пайплайн
`new -> plan -> approved -> implement -> review -> patch (optional loop) -> tests -> ready_pr -> done`

## Шаги
1. **new**
   - пользователь отправляет задачу/тикет
2. **plan**
   - оркестратор формирует краткий план:
     - summary
     - scope
     - change_plan (до 5–7 файлов на итерацию)
     - acceptance criteria
3. **approved**
   - пользователь подтверждает план (или правки)
4. **implement**
   - bridge передает задачу в Claude Code (режим implement)
5. **review**
   - Claude Code в режиме review проверяет изменения критически
6. **patch** (опционально)
   - фикс по замечаниям review
   - цикл `review <-> patch` ограничен (обычно 2–3 итерации)
7. **tests**
   - запуск тестов/чеклист тестирования
8. **ready_pr**
   - ветка `agent/*`, PR создан
9. **done**
   - итоговая сводка

## Правила остановки
- если после лимита итераций review остаются критические проблемы — эскалация пользователю
- если scope «распух» — декомпозиция на новые микро-задачи
