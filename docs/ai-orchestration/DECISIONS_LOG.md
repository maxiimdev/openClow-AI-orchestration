# DECISIONS_LOG

## 2026-03-02 — Baseline decisions (v1)

1. **Стратегия инструментов:**
   - не «или/или», а связка OpenClaw + Claude Code

2. **Роли в системе:**
   - OpenClaw = оркестратор
   - Claude Code = исполнитель кода

3. **Размещение Claude Code:**
   - первично использовать локальный Claude Code пользователя
   - не дублировать рабочую среду на сервере без необходимости

4. **Безопасность и ToS:**
   - не использовать извлеченный OAuth/Max токен Claude в OpenClaw
   - сторонние интеграции — только через допустимые каналы

5. **Git-политика:**
   - работа в отдельных ветках (`agent/*`)
   - PR-only, без прямого пуша в `main`

6. **Оркестрация:**
   - state-machine процесс с ограниченным review loop

7. **Экспертные агенты:**
   - вводятся как Expert Packs
   - старт с 3 ролей: frontend-lead, code-reviewer, test-engineer

8. **Токены и стоимость:**
   - cheap mode по умолчанию
   - deep mode только вручную
   - контекстная компрессия обязательна

9. **Документация:**
   - фиксировать архитектуру и правила в markdown
   - после стабилизации перенести в Obsidian как рабочую базу
