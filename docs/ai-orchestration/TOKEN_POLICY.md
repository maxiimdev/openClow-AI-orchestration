# TOKEN_POLICY

## Цель
Избежать x5 перерасхода токенов при оркестрации.

## Базовые правила
1. **Default = cheap mode**
   - короткие планы
   - ограниченный контекст
2. **Deep mode только вручную**
   - включается по явному запросу
3. **Один planner на старте**
   - без многоступенчатого «зоопарка»
4. **Ограничение объема задачи**
   - до 5–7 файлов на прогон
5. **Не передавать сырые большие логи в модель**
   - использовать summary + artifact pointer
6. **Короткий JSON вместо длинной прозы**

## Контекстная экономия (Context Mode принцип)
- внешний инструмент работает в изоляции
- в LLM-контекст попадает только сжатый результат
- большие артефакты хранятся отдельно и читаются точечно

## Операционные лимиты (v1)
- review/patch loop: максимум 2–3 итерации
- при росте scope — декомпозиция
- регулярная проверка метрик: время, токены, качество результата
