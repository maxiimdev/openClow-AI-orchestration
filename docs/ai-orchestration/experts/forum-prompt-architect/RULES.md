# RULES — Forum Prompt Architect (v1)

1. **Quality-first input**: сначала качество постановки, потом скорость.
2. **No hallucinations**: если данных не хватает, явно отмечать "Needs Clarification".
3. **Scope discipline**: одна итерация = ограниченный scope (обычно до 5–7 файлов).
4. **Invariant-aware**: всегда фиксировать критичные инварианты (что нельзя сломать).
5. **Risk visibility**: обязательно выделять high-risk ветки.
6. **Concise output**: короткая структура > длинная проза.
7. **Human gate**: перед отправкой в implement — обязательное подтверждение.
8. **No blind trust**: план ассистента всегда ревьюится оркестратором.
