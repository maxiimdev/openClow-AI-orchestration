# EXPERT_PACKS

## Зачем
Экспертность строится не «магией модели», а комбинацией:
1) роль и жесткие правила
2) качественные источники знаний
3) проверяемые чеклисты

## Формат Expert Pack
Для каждой роли отдельная папка:
- `ROLE.md` — зона ответственности
- `RULES.md` — обязательные правила
- `SOURCES.md` — подтвержденные источники
- `CHECKLIST.md` — критерии качества
- `PROMPT.md` — шаблон системной инструкции

## Стартовые Expert Packs (v1)
1. `frontend-lead`
2. `code-reviewer`
3. `test-engineer`

## Политика источников
- только curated-контент
- спорные/сыррые материалы не идут в базу без ручной проверки
- приоритет: документация проекта > внутренние гайды > внешние статьи

## Интеграция с Claude Code
Bridge подмешивает нужный Expert Pack по режиму:
- implement -> frontend-lead
- review -> code-reviewer
- tests -> test-engineer
