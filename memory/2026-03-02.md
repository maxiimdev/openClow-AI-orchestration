- Цель на 6 месяцев: стать сильным AI-assisted frontend специалистом, усилить процессы, подготовиться к поиску второй работы.

## 2026-03-02 — Детализация боли и приоритетов (интервью)

- Главный bottleneck: ручная проверка в браузере (network/console/store/flows), особенно на сложной регистрации и ролях.
- Класс проблем: баги на стыке состояния и бизнес-логики (персистентный стор, роли, токены/OTP, refresh, разный контекст регистрации).
- Были инциденты с репутационным ущербом из-за багов в регистрации/входе в ЛК.
- Частая проблема: неполные требования и слабая коммуникация с внешним бэкендом/аналитиками (не зона автоматизации ИИ сейчас).
- Тесты почти не покрывают критические сценарии; E2E/контрактные проверки нужны как обязательный контур.
- Приоритеты подтверждены:
  1) писать более качественный код без багов;
  2) автоматизировать флоу Jira -> plan -> code -> review/tests -> PR;
  3) подготовка к CRM-уровню сложности;
  4) упаковка резюме для второй работы.
