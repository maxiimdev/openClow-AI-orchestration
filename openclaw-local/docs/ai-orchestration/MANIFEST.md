# MANIFEST — AI Delivery System (v1)

## Зачем эта система
Сделать разработку предсказуемой: меньше хаоса, меньше багов, выше скорость и качество.

## Как она работает
1. Ты даешь задачу (тикет/идея).
2. Оркестратор структурирует её в Task Package.
3. Экспертный планировщик (Forum Prompt Architect) усиливает постановку.
4. Worker запускает Claude Code локально в безопасных границах.
5. Система проходит этапы: plan -> implement -> review -> tests -> result.
6. Ты получаешь статус и итог в Telegram.

## Почему это лучше ручного режима
- Четкая постановка вместо «сделай как-нибудь».
- Контроль рисков и инвариантов до реализации.
- Повторяемый процесс вместо разрозненных действий.
- Меньше ручной рутины, больше фокуса на решениях.

## Принципы
- Quality-first, но без перерасхода.
- Human-in-the-loop на ключевых шагах.
- Маленькие итерации и прозрачные статусы.
- Документация обновляется вместе с системой.

## Границы
- Никаких прямых изменений в main/master.
- Для сложных задач — architect-first.
- Для trivial задач — быстрый путь без лишней бюрократии.
